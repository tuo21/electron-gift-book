## 导入性能优化计划

### 问题
当前导入逻辑逐条插入记录，904 条数据需要 904 次 IPC 调用，导致 3 分钟的导入时间。

### 解决方案
修改 `App.vue` 中的 `handleImportData` 函数，使用已有的 `batchInsertRecords` 批量插入方法。

### 修改内容
**文件**: `src/App.vue`

将：
```ts
// 批量插入记录
if (dbRecords.length > 0) {
  for (const record of dbRecords) {
    const insertResponse = await window.db.insertRecord(record as any);
    if (!insertResponse.success) {
      alert('导入记录失败: ' + (insertResponse.error || '未知错误'));
      return;
    }
  }
}
```

改为：
```ts
// 批量插入记录
if (dbRecords.length > 0) {
  const insertResponse = await window.db.batchInsertRecords(dbRecords as any);
  if (!insertResponse.success) {
    alert('导入记录失败: ' + (insertResponse.error || '未知错误'));
    return;
  }
}
```

### 预期效果
- 904 条记录只需 1 次 IPC 调用
- 使用 SQLite 事务保证原子性
- 导入时间从 3 分钟降至几秒
